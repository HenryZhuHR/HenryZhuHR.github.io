# 自动驾驶模拟器 Carla

CARLA 是一个开源的自动驾驶(autonomous driving)模拟器，carla 的官方文档 CARLA Documentation非常详细

Carla是一个开源的自动驾驶系统模拟平台，它支持开发、训练和验证自动驾驶系统3。Carla提供了开放的数字资产（城市布局、建筑、车辆），它们是为这个目的而创建的，可以自由使用3。Carla还支持灵活的场景定义和传感器配置3。Carla可以从源代码构建，也可以使用预编译版本4。

如果只是使用 carla 的基本功能，可以只需要安装 Debian 版本，但是如果需要自定义场景、车型之类的功能需要借助虚幻引擎，这时候就需要从源码编译 carla

如果采用 Debian 方式安装，就可以简单启动。这里采用高画质、无显示器的启动方式

/opt/carla-simulator/CarlaUE4.sh \
	-quality-level=Epic \
	-RenderOffScreen

# carla 的使用流程

# 创建客户端

为了通过 Python API 操作 CARLA，需要通过一个端口(port)连接 CARLA服务器。客户端通过 client and world objects  控制仿真器

初始化一个客户端实例，建立起与服务器仿真器的的通信，参数分别为主机地址和端口，端口默认为 2000 。通常来说为了防止由于网络原因带来的阻塞，我们还可以设置这个客户端的超时时间。

import carla
client = carla.Client('localhost', 2000)
client.set_timeout(5.0)

成功导入carla库之后，需要先连接上 localhost:2000 端口，再按照下文的三个元素抽象层级的顺序进行编程。

  1. world
  2. blueprint
  3. actor

这使客户端设置为与在本地机器localhost上运行的CARLA服务器通信。或者，如果在单独的机器上运行客户端，可以使用网络机器的IP地址。第二个参数是端口号。默认情况下，CARLA服务器将在端口 2000 上运行，如有必要，您可以在启动CARLA时在设置中更改此操作。

# 世界 world

可以获取当前的世界，也可以加载指定的世界地图

world = client.get_world()
world = client.load_world('Town04')

carla 自带的地图 Town01~ Town07

# Actors 行为体

CARLA 中的 Actors 是在模拟器内执行动作的元素，它们也可以影响其他的 Actors。CARLA 中的 Actors 包括车辆 (vehicles)、行人 (walkers)、传感器 (sensors)、交通信号 (traffic signs)、交通灯 (traffic lights)和观察者 (spectator)。充分了解如何操作它们至关重要(crucial)。

针对 Actors 的操作主要有：产生 (spawning)、销毁 (destruction)、类型 (types) 以及管理 (manage)

这些布局允许用户顺利地将新参与者纳入模拟中。它们已经制作了带有动画和一系列属性的模型。其中一些是可修改的，而另一些则不可修改。除其他外，这些属性包括车辆颜色、激光雷达传感器中的通道量、步行者的速度等等。

# blueprints

世界包含了蓝图 (blueprints) 列表，我们可以使用蓝图在仿真器中创建行为 (actors) 蓝图列表类 carla.BlueprintLibrary 由 carla.ActorBlueprint 元素的列表组成，它是能够提供访问操作的世界对象。

blueprint_library = world.get_blueprint_library()

返回类型实际上是 carla.libcarla.BlueprintLibrary 而不是列表，但是可以将其完全看成一个列表进行操作

蓝图列表所包含的内容如下，每一个都是 carla.libcarla.ActorBlueprint 类实例

ActorBlueprint(id=static.prop.mailbox,tags=[mailbox, prop, static])
ActorBlueprint(id=walker.pedestrian.0018,tags=[0018, pedestrian, walker])
ActorBlueprint(id=sensor.other.radar,tags=[radar, other, sensor])
ActorBlueprint(id=vehicle.micro.microlino,tags=[microlino, micro, vehicle])

我们通过 id 对其进行区别

得到蓝图列表后，通过 find() 函数，按照 id 进行查找；或者通过 filter() 函数，对其进行过滤

walker_bp = blueprint_library.filter('walker.pedestrian.0018')
vehicle_bp = blueprint_library.filter('vehicle')
vehicle_bp_rd = random.choice(blueprint_library.filter('vehicle.*.*'))

我们选择特斯拉的 Model3 ，并且可以获取和设置 Actors 的属性 attribute

model3_bp = world.get_blueprint_library().find('vehicle.tesla.model3')
model3_bp.get_attribute('number_of_wheels')
model3_bp.set_attribute('color', '255,255,255')

属性设置 https://carla.readthedocs.io/en/latest/bp_library/

# Actor 的生命周期

# Spawn 产生

世界物体负责产生(spawn) Actor 并跟踪这些 Actor。产生只需要 blueprint 和 carla.Transform，描述 Actor 的位置(location)和旋转(rotation)。

世界内包含了两种不同的方法产生 Actor，返回值均为 carla.Actor

  • spawn_actor() ：如果产生失败将产生异常
  • try_spawn_actor() 如果产生失败将返回 None

上述两种方法输入参数都是相同的： spawn_actor(self, blueprint, transform, attachto=None, attachment=Rigid)

  • blueprint (carla.ActorBlueprint)：根据蓝图生成 actor
  • transform (carla.Transform)：包含演员将生成的位置和方向
  • attach_to (carla.Actor)：（通常是传感器）安装在哪个 actor 上，默认为None，参考 传感器生成
  • attachment (carla.AttachmentType)：安装方式，参考 传感器安装类型
# 设置出生点
spawn_transform = carla.Transform(
	location=carla.Location(x=50,y=100 ,z=10 ),
	rotation=carla.Rotation(pitch=0.0, yaw=0.0, roll=0.0)
)
# 获取随机出生点
spawn_transform = np.random.choice(world.get_map().get_spawn_points())
# 生成 model3 模型
model3_actor = world.spawn_actor(blueprint, transform)

注意 PythonAPI 中 carla.Location 的单位是 m，在 UE4 中是 cm。 CARLA 使用的是 Unreal Engine 坐标系 。carla.Rotation 结构为 (pitch, yaw, roll), 和 Unreal Engine Editor中 (roll, pitch, yaw) 不同。 转至 坐标系 查看更多。

值得注意的是，若在指定位置 location 与其他 Actor 发生碰撞时，Actor 不会产生。无论这是否发生在静态对象或其他 Actor 者身上。使用下列的方法获取位置，避免这些不必要的产生碰撞。

  • 对于车辆，使用 map.get_spawn_points() ，返回一个推荐生成点的列表。生成点的列表的元素类型为 carla.Transform ，可以获取到出生点的位置和旋转信息
spawn_points = world.get_map().get_spawn_points()

for sp in spawn_points:
	print(sp.location.x, sp.location.y, sp.location.z)
	print(sp.rotation.pitch, sp.rotation.yaw, sp.rotation.roll)
  • 对于行人 world.get_random_location() ，但会人行道上的随机点
spawn_point = carla.Transform()
spawn_point.location = world.get_random_location_from_navigation()

在产生时，Actor 可以安装 (attach) 在另一个 Actor 身上，这时候 Actor 将跟随他们所安装的 parent Acotor，这对传感器特别有用。这种安装可以是刚性的（可以获取精确的数据），也可以根据其 parent Acotor 进行轻松移动，安装方式由 carla.AttachmentType定义

attach 原意为装上,贴上,系上;使依附;使附属，这里将 attach 翻译成安装

将相机刚性地固定在车辆上，因此它们的相对位置保持固定。

camera = world.spawn_actor(camera_bp, relative_transform, attach_to=model3_actor, carla.AttachmentType.Rigid)

当安装在其他的 Actor，必须要提供相对于 parent Actor 的转换。 When spawning attached actors, the transform provided must be relative to the parent actor.

# Handling 操纵

大部分 carla.Actor 都提供了get() 和 set() 方法管理地图中的 actors

print(model3_actor.get_acceleration())
print(model3_actor.get_velocity())

location = model3_actor.get_location()
location.z += 10.0
model3_actor.set_location(location)

Actor 的物理仿真可以被禁止，并将其冻结在原地

model3_actor.set_simulate_physics(False)

大多数方法采用异步模式向模拟器发送请求，模拟器每次更新解析它们的时间有限。频繁的 set() 方法淹没模拟器将累积显著滞后

设置为自动驾驶

model3_actor.set_autopilot(False)

# Destruction 销毁

当 Python 脚本结束之后，Actors 会被隐式地销毁

destroyed_sucessfully = model3_actor.destroy() # Returns True if successful

销毁 Actor 会将模拟器锁住，直到进程结束

Actors 的类型主要分为Sensors 传感器、Spectator 观察者

# Sensors 传感器类型

传感器 (Sensors) 能够产生数据流。

# Spectator 观察者类型

Spectator 由 Unreal Engine 放置，以提供游戏中的视角，它可用于移动模拟器窗口的视角。以下示例将移动观众演员，将视图指向所需的车辆。

spectator = world.get_spectator()
transform = model3_actor.get_transform()
spectator.set_transform(carla.Transform(transform.location + carla.Location(z=50), carla.Rotation(pitch=-90)))

# 传感器与数据 Sensor and Data

传感器类 carla.Sensor 定义了一种能够从周围环境获取数据的特殊 Actor。

与所有其他Actor一样，找到蓝图并设置特定的属性。这在处理传感器时至关重要。他们的属性将决定获得的结果。这些在传感器参考中进行了详细说明。

设置一个 RGB 摄像头如下

# Find the blueprint of the sensor.
camera_bp = blueprint_library.find('sensor.camera.rgb')
# Modify the attributes of the blueprint to set image resolution and field of view.
camera_bp.set_attribute('image_size_x', '1920')
camera_bp.set_attribute('image_size_y', '1080')
camera_bp.set_attribute('fov', '90')
camera_bp.set_attribute('sensor_tick', '1.0') # Set the time in seconds between sensor captures

可以设置相机中心一直指向车

def get_rotation_by_center_actor(x=0., y=0., z=0.):
    x = x + 1e-9
    y = y + 1e-9
    # Step 1:  Since x can be negtiva, you do not change the symbol
    yaw = 180 if x > 0 else 0 + np.arctan(y / x) * (180 / np.pi)
    # Step 2:  pith is always negitive to see down
    pitch = -np.arctan(z / (np.sqrt(x * x + y * y))) * (180 / np.pi)
    return (pitch, yaw, roll := 0.)

# 传感器生成

生成传感器最重要的参数是 attach_to 和 attachment_type 。

attach_to 将决定传感器安装在哪一个 Actor 上，通常是车辆 vehicle

attachment_type 将决定传感器会如何根据车辆位置进行更新，安装方式由 carla.AttachmentType定义，有如下三种安装方式：

  • Rigid 硬安装。传感器将严格根据所安装的 Actor 的位置移动，这种安装方式适合于从模拟器中获取数据的。
  • SpringArm 弹簧臂安装。传感器通过较小的加速度和减速来缓解移动，这使得在更新相机位置时，传感器能够平稳运动，避免跳跃，这种方式适用于在模拟器中录制视频。这种安装方式来源于虚幻引擎中的弹簧臂组件
  • SpringArmGhost SpringArmGhost附件。像前一个一样，但没有进行碰撞测试，因此相机或传感器可以穿过墙壁或其他几何形状。
transform = carla.Transform(carla.Location(x=0.8, z=1.7))
sensor = world.spawn_actor(
	blueprint, 
	transform, 
	attach_to=my_vehicle, 
	attachment_type=carla.AttachmentType.SpringArm
	)

当传感器被安装在车辆上时，位置必须相对于 parent Actor。

# 监听传感器

每一个传感器都有 listen() 方法，参数是一个回调函数 callback ，该方法用于处理传感器获取的数据

# do_something() will be called each time a new image is generated by the camera. 
sensor.listen(lambda data: do_something(data)) 
... 
# This collision sensor would print everytime a collision is detected. 
def callback(event): 
	for actor_id in event:
		vehicle = world_ref().get_actor(actor_id) 
		print('Vehicle too close: %s' % vehicle.type_id) 
sensor02.listen(callback)

每次收到新的测量结果时，传感器都会调用的功能。此函数需要包含对象类型carla.SensorData的参数才能使用。

大多数传感器数据对象都有将信息保存到磁盘的功能。这将允许它在其他环境中使用。

传感器类型的传感器数据差异很大。看看传感器参考资料，以获得详细的解释。 然而，它们总是被标记为一些基本信息。

Sensor data 属性描述类型
frame 测量时的帧序号int
timestamp 自 episode 开始以来，以模拟秒为单位的测量时间戳double
transform测量时传感器的世界参考carla.Transform

is_listening is a sensor attribute that enables/disables data listening at will.
sensor_tick is a blueprint attribute that sets the simulation time between data received.

# 坐标系

CARLA使用 Unreal Engine 坐标系统，这是一个 Z-up left-handed 系统。

构造函数方法遵循特定的声明顺序：(pitch, yaw, roll)，对应于(Y-rotation,Z-rotation,X-rotation)。 音高、偏航、滚动 Y-旋转、Z-旋转、X-旋转

Warning：在 CARLA (pitch, yaw, roll) 和 Unreal Engine Edito (roll, pitch, yaw) 中旋转坐标的声明顺序不同。当从源码构建时，不要混淆轴的旋转。
