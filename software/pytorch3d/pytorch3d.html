# 坐标转换约定

渲染需要在几个不同的坐标系之间进行转换：「世界空间」、「视图/相机空间」、「NDC(Normalized Device Coordinates)空间」和「屏幕空间」。

在每一次转换中了解「相机的位置」、「+X、+Y、+Z轴的对齐方式」以及「值的范围」都很重要。下图概述了PyTorch3D使用的约定。

  • PyTorch3D中的NDC坐标系与OpenGL中的左手NDC坐标系统相比是右手的（投影矩阵切换惯用手）。

# 相机坐标系统

有 4 个坐标系统：

  • 「世界坐标系」：对象/场景所生活的系统

  • 「相机坐标系」：原点在像平面上，Z轴垂直于像平面。在 PyTorch3D 中，假定 +X 指向左侧，+Y 指向上方，+Z 指向屏幕外。通过旋转矩阵（R）和平移矩阵（T），将视角从世界坐标系转化至相机坐标系

  • 「NDC」：这是将对象/场景的渲染部分限制在体中的标准化坐标系。也称为视图体积。对于正方形图像，根据PyTorch3D约定，(+1, +1, znear) 是体的左上角(近角)，(-1, -1, zfar) 是体的右下角(远角)。对于非正方形图像，XY中体积长度最小的边在 [-1, 1] 的范围内，而较大的边在 [s, s] 的范围内。其中s是纵横比，s>1（较大的边除以较小的边）。从视图到NDC的转换发生在应用相机投影矩阵（P）之后。

  • 「屏幕坐标系」：这是视图体积的另一种表示方式，XY坐标在像素空间而不是标准化空间中定义。(0, 0)是左上像素的左上角，(W, H)是右下像素的右下角。

# 定义相机

PyTorch3D 中的相机将对象/场景从世界转换为视图，方法是首先将对象/场景转换为视图（通过转换 R 和 T ），然后通过投影矩阵 P = K[R | T] 将3D对象/场景投影到标准化空间，其中 K 是内参矩阵。

K 中的相机参数定义归一化空间。如果用户在 ND C空间中定义相机参数，则变换项目指向 NDC 。如果相机参数在屏幕空间中定义，则变换后的点在屏幕空间中。
