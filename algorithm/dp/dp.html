# 动态规划

动态规划 (Dynamic Programming, DP) ，如果某一问题有很多重叠子问题，使用动态规划是最有效的。

所以动态规划中每一个状态一定是由上一个状态推导出来的，这一点就区分于贪心，贪心没有状态推导，而是从局部直接选最优的，

  • 动规 是由前一个状态推导出来的
  • 贪心 是局部直接选最优的

# 🎒 01背包问题

01背包问题如下：有 n 件物品和一个最多能背重量为 c 的背包。第 i 件物品的重量是 wi​ ，价值是 vi​ 。每件物品只能用一次，求解将哪些物品装入背包里物品价值总和最大。

可以定义一个递推公式 f(i,j) ，其中 i=1,2,...,n;j=0,1,...,c ，表示前 i 件物品恰放入一个容量为 j 的背包可以获得的最大价值。则其状态转移方程可以从两个角度考虑：

  • 不放入第 i 件物品，那么当前的最大价值 f(i,j) 就是 f(i−1,j) 。其实就是当物品 i 的重量大于背包 j 的重量时，物品 i 无法放进背包中，所以背包内的价值依然和前面相同。

  • 放入第 i 件物品，那么 f(i,j) 可以从 f(i−1,j−wi​)+vi​ 推出。因为放入第 i 件物品，背包剩余容量为 j−wi​ ，那么最大价值由 f(i−1,j−wi​) 和第 i 件物品的价值 vi​ 组成

因此，状态转移方程为

f(i,j)={f(i−1,j)f(i−1,j−wi​)+vi​ ​,j<wi​,j≥wi​​

如果第 i 件物品的重量是 weight[i] ，价值是value[i]，那么状态转移方程可以写成：

dp[i][j]=max(
  dp[i-1][j],                       // 0 不放
  dp[i-1][j - weight[i]] + value[i] // 1 放入
)

可以看出 i 是由 i-1 推导出来，那么 i=0 的时候就一定要初始化，因为背包容量为 0 时，无法放入任何物品，因此价值为 0

vector<vector<int>> dp(n, vector<int>(bagweight + 1, 0));
dp[0][j] = 0;
for(int i = 1; i < weight.size(); i++) { // 遍历物品
  for(int j = 0; j <= bagweight; j++) {    // 遍历背包容量
    if (j < weight[i])        // 如果背包容量小于物品重量，
      dp[i][j] = dp[i-1][j];  // 那么物品i无法放进背包中，直接复制上一层
    else                      // 否则，就要考虑放入物品i的情况
      dp[i][j] = max(
        dp[i-1][j],                     // 不放
        dp[i-1][j-weight[i]] + value[i] // 放入
        // 放进去的话，考虑上一层背包容量减去物品i的重量，加上物品i的价值
      );

  }
}

采用一个二维数组，因此空间复杂度为 O(nc) ，其中 n 是物品个数， c 是背包容量。时间复杂度为 O(nc)

从状态转移方程可以看出，dp[i][j] 的更新只与 dp[i-1][*] 有关，其实可以发现如果把 dp[i-1] 那一层拷贝到 dp[i] 上，表达式完全可以是：

dp[i][j] = max(dp[i][j], dp[i][j - weight[i]] + value[i]);
  • dp 数组在推导的时候一定是取价值最大的数，如果题目给的价值都是正整数那么非0下标都初始化为0就可以了。
  • 遍历的时候一定需要倒序，因为二维数组用到 i-1 ，所以必须倒叙遍历背包，不然 i-1 会被覆盖

倒序遍历是为了保证物品i只被放入一次！。但如果一旦正序遍历了，那么物品0就会被重复加入多次！

可以参考 “背包问题（01背包+滚动数组+完全背包）”，考虑 j 和 2j 的递推: dp[ j] = max(dp[ j], dp[ j-weight[i]]+value[i]);：

dp[ j] = max(dp[ j], dp[ j - weight[i]] + value[i]);
dp[2j] = max(dp[2j], dp[2j - weight[i]] + value[i]);

// 当 j = weight[i] 这个特殊情况
dp[ j] = max(dp[ j], dp[0] + value[i]);
dp[2j] = max(dp[2j], dp[j] + value[i]);
// 可以看到， weight[i] 被重复放进了两次

优化后的代码如下：

vector<int> dp(bagWeight + 1, 0);
for(int i = 0; i < weight.size(); i++) {      // 遍历物品
  for(int j = bagWeight; j >= weight[i]; j--) { // 遍历背包容量(倒序)
      dp[j] = max(
        dp[j],                       // 不放
        dp[j - weight[i]] + value[i] // 放入
      );
      // 如果方案和，那么就把不放和放的加起来
      // dp[j] = dp[j] + dp[j - weight[i]] (+ value[i]);
  }
}

01 背包中组合数问题

/**
 * 当 1≤i≤n时，对于数组 nums 中的第 i，遍历 0≤j≤neg，计算 dp[i][j]：
 * 如果 j<num，则不能选 num，  此时有 dp[i][j]=dp[i-1][j]；
 * 如果 j≥num，
 *  - 不选  num 方案数是 dp[i-1][j]
 *  - 选    num 方案数是 dp[i-1][j-num]，
 * 此时有 dp[i][j] = dp[i-1][j] + dp[i-1][j-num]
 *
 * 装满背包有几种方法的递推公式
 */
dp[j] = dp[j] + dp[j - nums[i]];

/**
 * 表示 已经 有 nums[i] 时，有 dp[j - nums[i]] 种方法凑成 j
 * 例如：dp[j]，j 为5，
 * nums[i] = 1 , 有 dp[5-1] = dp[4] 种方法凑成 5
 * nums[i] = 2 , 有 dp[5-2] = dp[3] 种方法凑成 5
 * ...
 * dp[j] 定义就是凑成和为 j 有多少种方案
 * 初始为 dp[0] = 1 ，因为不取任何数，和就是 0 (唯一方案)

# 📈 买卖股票问题

使用动态规划的方法，维护在股市中每一天结束后可以获得的「累计最大收益」，并以此进行状态转移

买卖股票是一个状态机(s_k[i]状态由 i-1 时刻状态转移得到)

  • 边界条件:
    • s0[0] = 0 : 第0天不持有股票，那么利润为 0
    • s1[0] = INT_MIN: 第0天持有股票是不合法的，所以初始化为 −∞ ，这样取 max() 就会转化成合法状态

参考算法:

// 你最多可以完成 k 次交易
int maxProfit(vector<int>& prices, const int& k)
{
    vector<int> s1(k + 1, INT_MIN);// 滚动数组
    vector<int> s0(k + 1, 0);      // 滚动数组

    for (const auto& price : prices){
        for (int i = 1; i <= k; i++){
            s1[i] = max(          // 今天 持有 状态
                s1[i],            // 0 不买不卖（状态保持）
                s0[i - 1] - price // 1 买入 （只能由s0[i-1]转移得到）
            );
            s0[i] = max(          // 今天 不持 状态
                s0[i],            // 0 不持有 （状态保持）
                s1[i] + price     // 1 卖出 （只能由s1[i-1]转移得到）
                // 疑惑的是，这里用到了 s1[i] ，是刚才更新的状态
                // 实际上表示，今天买入后又卖出了，相当于没有交易
            );
        }
    }
    return no_hold[k];
}
