C++ 智能指针

HenryZhu
2023-09-06
13 min

# 智能指针

背景：

  • 指针可能导致「内存泄露(memory leak)」和「内存非法访问」
  • 「动态内存管理」 和 「垃圾回收（Gabage Collector）机制」
  • C++是面向内存编程，Java是面向数据结构编程。
  • 「C++ 常见的内存错误」： coredump、segmentfault 之类的内存问题，使用指针也会出现各种问题，比如:
    • 「野指针」：未初始化或已经被释放的指针被称为野指针
    • 「空指针」：指向空地址的指针被称为空指针
    • 「内存泄漏」：如果在使用完动态分配的内存后忘记释放，就会造成内存泄漏，长时间运行的程序可能会消耗大量内存。
    • 「悬空指针」：指向已经释放的内存的指针被称为悬空指针
    • 「内存泄漏和悬空指针的混合」：在一些情况下，由于内存泄漏和悬空指针共同存在，程序可能会出现异常行为。

智能指针是 C++ 11 引入的一个新特性，它的目的是解决 C++ 中的内存泄漏问题。智能指针是一个模板类，它的实例对象的行为类似于指针，但是它能够自动释放所指向的对象。

智能指针的核心思想就是 RAII 机制

C++11 引入了 3 个智能指针类型 (#include <memory>)：

  • std::unique_ptr<T>: 独占资源所有权的指针。
  • std::shared_ptr<T>: 共享资源所有权的指针。
  • std::weak_ptr<T>: 共享资源的观察者，需要和 std::shared_ptr 一起使用，不影响资源的生命周期。

# unique_ptr

std::unique_ptr 是一个独占所有权的智能指针，保证指向的内存只能由一个 unique_ptr 拥有，不能共享所有权。

超出作用域时，它所指向的内存会自动释放。

因为众所周知的原因， 直到 C++ 14 才引入了 std::make_unique 函数，用于创建 unique_ptr 对象。

#include <memory>
#include <iostream>
int main() {
    // std::unique_ptr<int> ptr(new int(10));   // 不推荐使用
    std::unique_ptr<int> ptr = std::make_unique<int>(10);   // 使用 make_unique 创建对象
    std::cout << *ptr << std::endl; // 输出10
    // unique_ptr在超出作用域时自动释放所拥有的内存
    return 0;
}

# shared_ptr

std::shared_ptr 是一个「共享所有权」的智能指针，它允许多个 shared_ptr 指向同一个对象，并使用引用计数来记录所指向对象的 shared_ptr 的个数。

当最后一个 shared_ptr 超出作用域时（引用计数为0），所指向的内存才会被自动释放。

使用 shared_ptr 时，需要注意避免循环引用。

C++ 11 同时引入了 std::make_shared 函数，用于创建 shared_ptr 对象，它可以减少内存分配的次数，提高性能。

#include <memory>
#include <iostream>
int main() {
    // std::shared_ptr<int> ptr1(new int(10)); // 不推荐使用
    std::shared_ptr<int> ptr1 = std::make_shared<int>(10); // 使用 make_shared 创建对象
    {
        std::shared_ptr<int> ptr2 = ptr1; // 通过拷贝构造函数创建一个新的 shared_ptr 
        // 引用计数为 2
        std::cout << *ptr1 << " " << *ptr2 << std::endl; // 输出 10 10
    }
    // ptr2 超出作用域时，所指向的内存不会被释放，因为此时 ptr1 仍然持有对该内存的引用
    return 0;
}

# 智能指针的场景

通常来说，业务代码使用智能指针，而底层数据结构就没有必要使用智能指针了

# 需要使用指南指针的场景

  • 有多个任务同时使用一个资源的时候，需要在全部任务都结束的时候释放资源。这种情况下，可以使用 shared_ptr。因为不可能再开一个任务去监视其它任务的结束，所以只能使用引用计数的方式来判断资源是否可以释放。

  • 需要的资源可能有效或者在其他地方被清理，当我们需要的时候，需要检查资源是否有效。这种情况下，可以使用 weak_ptr。

    因为 weak_ptr 不会增加引用计数，所以即使资源已经被清理，weak_ptr 也不会知道。所以我们需要在使用 weak_ptr 的时候，先检查资源是否有效。

  • 一个流程（可能是一个函数中）有多个退出的位置 (return 或者 异常退出)，这时候使用 unique_ptr 可以避免在每一个退出位置都释放资源

# 不要使用指南指针的场景

  • 不要在单链表中使用智能指针，会导致内存泄漏（链表稍微长一点，就会栈移除）。因为单链表中的节点是通过指针连接起来的，如果使用智能指针，那么每个节点都会有一个引用计数，这样就会导致节点永远无法被释放。

  • 不要再双向链表中使用 shared_ptr，因为双向链表中的节点有两个指针，如果使用 shared_ptr，那么每个节点都会有两个引用计数，这样就会导致节点永远无法被释放。

# C++ 内存管理

# 指针

指针本身也是一个变量，其存储「它所指向变量的地址」，指针也有自己的地址

  1. 解引用
float f = 1.0;
short c = *(short*)&f; 
// &f       取首位地址 
// (short*) 转化为2字节的指针
// *        取2字节的值

从内存层面来说，f 没有发生任何改变：

// float f = 1.0 ; 的内存布局 4 Byte (32 bit)
// float :  1 bit 符号位 , 8 bit 指数位 , 23 bit 尾数位 (7位有效数字)
| 0000 0000 | 1000 0000 | 0000 0000 | 0000 0000 |  // (大端为例)
|    0x00   |    0x7f   |   0x00    |   0x00    |  // f
|    0x00   |    0x7f   |                          // c

指数位: 用于存储科学计数法中的指数数据，并且采用移位存储

如果反过来，那么就可能发生 coredump(堆芯转储)，也就是访存失败。即使没有 coredump，这种也会破坏这块内存原有的值

short c = 1;
float f = *(float*)&c;
  1. 指针数组

实际上可以看做 address[offset]，address 为起始地址，offset 为偏移量: address + sizeof(int) * offset。 这是汇编中寻址方式的一种：基址变址寻址 (MOV AX , [ BX + SI ])。

访问二维数组 array[n][m] 中的 array[a][b]: array + (m * a + b)

  1. void 指针

在这些情况下，void 表达的意思就是没有返回值或者参数为空。但是对于 void 型指针却表示通用指针，可以用来存放任何数据类型的引用。

void func();
int func1(void);

void 指针最大的用处就是在 C 语言中实现泛型编程，因为任何指针都可以被赋给 void 指针，void 指针也可以被转换回原来的指针类型， 并且这个过程指针实际所指向的地址并不会发生变化。

void 还可以用来实现 C 语言中的多态，这是一个挺好玩的东西。

但是 不能对 void 指针解引用

  1. 指针声明
int (*(*p)[5])(int);
  • 指针数组: int *p[3] 。 p 先与 [] 结合，因为其优先级比高，所以 p 是一个数组。然后再与 * 结合，说明数组里的元素是指针类型。之后再与 int 结合，说明指针所指向的内容的类型是整型的，所以 p 是一个指向 int 的指针数组。
  • 数组指针: int (*p)[3] 。 指向 int 数组(大小为3)的指针
  • 函数声明: int p(int);
  • 函数指针: int (*p)(int);

# 内存分区

程序运行时，代码、数据等都存放在不同的内存区域，这些内存区域从逻辑上做了划分，大概以下几个区域：

  • 代码区(Code Segment): .text 段。存放二进制代码，只读
  • 全局/静态存储区(Global/Static Storage): .bss 和 .data 段
    • 以前在 C 语言中全局变量又分为初始化的和未初始化的，分别放在上面图中的 .bss 和 .data 段，但在 C++里面没有这个区分了，他们共同占用同一块内存区，就叫做全局存储区。
  • 栈区(Stack): 栈区用于存储函数调用时的局部变量、函数参数以及返回地址。
  • 堆区(Heap): 动态内存分配的区域，当使用 new 或者 malloc 分配内存时，分配的内存块就位于堆区。需要手动释放这些内存，否则可能导致内存泄漏
  • 常量区(Constant Storage): 字符串字面量和其他编译时常量。这个区域通常也是只读的

# 指针和引用区别

都是用于间接访问变量，区别：

  1. 「指针」 是一个变量，保存另一个变量的地址； 「引用」 是另一个变量的别名，与原变量共享内存地址。
  2. 「指针」 可以被重新赋值，指向不同的变量；「引用」 在初始化后不能更改，始终指向同一个变量。 「指针可以运算，这也是指针最危险的地方」
  3. 「指针」 可以为 nullptr，表示不指向任何变量；「引用」 必须绑定到一个变量，不能为 nullptr。
  4. 使用 「指针」 需要对其进行解引用以获取或修改其指向的变量的值；「引用」 可以直接使用，无需解引用。

汇编层面没有区别，引用会被c++编译器当做const指针来进行操作。

# 指针传递、值传递、引用传递

  1. 值传递是将实参的值传递给形参
  2. 引用传递是将实参的引用传递给形参。函数内对形参的修改会影响到实参
  3. 指针传递是将实参的地址传递给形参。函数内对形参的修改会影响到实参。
  • 堆 是指动态分配内存的一块区域，一般由程序员手动分配(new malloc)
  • 栈 是则是由编译器自动分配释放，用于存放函数的参数值，局部变量等
  • 函数调用栈:一个函数需要在内存上存储：
    • 函数的返回地址、函数的参数、局部变量、临时变量等。这些数据都是通过栈来分配的，每调用一个函数，就会将这些数据压入栈中，函数返回时，再将这些数据弹出栈。
    • 当多个函数相互调用的时候，就还需要机制来保证它们顺利的返回和恢复主调函数的栈结构信息。那这部分就包括返回地址、ebp寄存器（基址指针寄存器，指向当前堆栈底部） 以及其它需要保存的寄存器。

那么 func_a(a, &b) 在函数调用栈上:

  • a 的「值」拷贝到 func_a 函数参数 a 位置
  • b 的「地址」拷贝到 func_a 函数参数 b 位置
  • 本质都是拷贝

# RAII

资源获取即初始化（Resource Acquisition Is Initialization，简称 RAII）是一种 C++ 编程技术，它将在使用前获取的资源的生命周期与某个对象的生命周期「绑定」在一起。包括：

  • 分配的堆内存、执行线程、打开的套接字、打开的文件、锁定的互斥量、磁盘空间、数据库连接等有限资源

核心思想： 利用栈上局部变量的自动析构来保证资源一定会被释放。

一般设计实现一个 RAII 类需要四个步骤:

  1. 设计一个类封装资源，资源可以是内存、文件、socket、锁等等一切
  2. 在构造函数中执行资源的初始化，比如申请内存、打开文件、申请锁
  3. 在析构函数中执行销毁操作，比如释放内存、关闭文件、释放锁
  4. 使用时声明一个该对象的类，一般在你希望的作用域声明即可，比如在函数开始，或者作为类的成员变量

在 C++ 中，可以使用 RAII 思想来包装 mutex，确保在多线程编程中始终能安全锁定和解锁互斥量，这个用得非常多，可以在很多开源项目中看到这样的包装：

展开查看源码
#include <iostream>
#include <mutex>
#include <thread>

class LockGuard {
private:
    std::mutex &mutex_;
public:
    explicit LockGuard(std::mutex &mtx) : mutex_(mtx) {
        mutex_.lock();
    }
    ~LockGuard() {
        mutex_.unlock();
    }
    // 禁止复制
    LockGuard(const LockGuard &) = delete;
    LockGuard &operator=(const LockGuard &) = delete;
};

std::mutex mtx;       // 互斥量
int shared_data = 0;  // 多线程操作的变量
void increment() {
    for (int i = 0; i < 10000; ++i) {
        // 申请锁
        LockGuard lock(mtx);
        ++shared_data;
        // 作用域结束后会析构 然后释放锁
    }
}
int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    t1.join();
    t2.join();
    std::cout << "Shared data: " << shared_data << std::endl;
    return 0;
}

上面定义了一个 LockGuard 类，该类在构造函数中接收一个互斥量（mutex）引用并对其进行锁定，在析构函数中对互斥量进行解锁。

这样，我们可以将互斥量传递给 LockGuard 对象，并在需要保护的代码块内创建该对象，确保在执行保护代码期间始终正确锁定和解锁互斥量。
