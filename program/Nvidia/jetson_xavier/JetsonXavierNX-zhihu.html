# Nvidia Xavier NX 安装系统和基本配置

Nvidia Xavier NX 的官方文档可以参考如下：

  • Jetson Download Center 中的 Jetson Xavier NX Developer Kit User Guide
  • NVIDIA Jetson Guide

# 安装 Jetson Jetpack

如果是 EMMC 版本的 Jetson Xavier NX ，需要下载安装 SDK Manager进行 JetPack 系统安装

官方安装教程 NVIDIA SDK Manager Documentation

通过 SDK Manager 安装 JetPack 需要让 Jetson Xavier NX 处于“强制恢复模式”(Force Recovery mode)，给 Xavier 上电前使用杜邦线将 FC REC 引脚(9) 和 GND 引脚(10) 短接即可

引脚可能因型号而异，所以建议翻到板子背面查看引脚名称

# 步骤一：开发环境配置 (STEP 01: DEVELOPMENT ENVIRONMEN)

打开 SDK Manager ，根据提示登录 NVIDIA 账号。

  • HARDWARE CONFIGURATION: 用 micro—USB 连接主机(Host)和目标硬件(Target Hardware)，软件会自动识别已经连接上的开发板型号，确保处于“强制恢复模式”(Force Recovery mode)：
  • TARGET OPREATING SYSTEM: 操作系统选择，这里选择了 JetPack 5.1
  • ADDITIONAL SDKS: 可选是否安装 DeepStream 6.2

# 步骤二：选择组件 (STEP 02: DETAILS AND LICENSE)

这里选择好需要安装的 components，默认是全选了，如果 EMMC 上空间不够，那就可以取消 Jetson SDK Components ，等待系统安装好后，安装至 NVME 中。

选择好后，选择接受协议，点击 CONTINUE to STEP 03 进行下一步设置

# 步骤三：系统设置 (STEP 03: SETUP PROCESS)

这一步的设置比较重要，需要修改如下的配置：

  1. 修改为 Manual Setup - Jetson Xavier NX
  2. 设置进入系统的用户名和密码，这里建议设置为 xavier/xavier 即可
  3. Storage Device 选择 EMMC/SD Card (default) ，之后会将系统迁移至 NVME 中进行扩容

然后进入漫长的等待，安装到

大约进行到一半，会弹出一个安装 SDK components 的确认选项，这里如果可以成功安装就 install ，否则可以跳过

暂时没有成功安装这一步，但是貌似不影响后续的系统安装

然后继续漫长的等待，直到结束

# 步骤四：启动 Jetson Xavier

进入系统后

sudo apt -y update
sudo apt -y upgrade

安装出现 Unable to fetch some archives, maybe run apt-get update or try with --fix-missing? 则执行 sudo apt -y update --fix-missing 再继续

# 步骤五：系统盘迁移

由于 Jetson Xavier 自带的 EMMC 较小，只有 16G，所以考虑将 NVMe SSD 硬盘作为系统盘（rootfs和用户区），系统的启动引导依然是通过 EMMC，比如升级设备树dtb 还是在 EMMC中。

准备M.2 SSD（NX支持NVME接口）

  1. 打开 Ubuntu 自带 Disks 工具，点击右上角选择 Format Disk 并将其格式化为 GPT 格式

  1. 格式化后进行磁盘分区，建议分成主分区和一个16GB的交换文件区；并在下一步进行命名

  1. 点击下方 ‘三角按钮’ 将 SSD 挂载(mount)到固定目录如 /media/nvidia/<name> (<name> 是刚刚的命名)

完成后将系统盘迁移至 SSD 中

git clone https://github.com/jetsonhacks/rootOnNVMe.git
cd rootOnNVMe
./copy-rootfs-ssd.sh  # 复制到 ssd
./setup-service.sh    # ssd 设置为启动盘
sudo reboot

需要在 Xavier 查看 jetpack 版本，并记住版本号

sudo apt-cache show nvidia-jetpack

继续使用 micro-USB 连接连接 Host 主机和 Xavier ，然后在主机上运行 SDKManager

# 功耗模式

在 Supported Modes and Power Efficiency 中有详细介绍了 Jetson Xavier NX 的9种功耗模式

# 功耗模式设置

命令 nvpmodel 用于查看和修改功耗模式，各个功耗模式配置在 /etc/nvpmodel.conf 中

查看 Xavier 的功耗模式，默认模式为 5(MODE_10W)

$ sudo nvpmodel --query
$ sudo /usr/sbin/nvpmodel -q
NV Power Mode: MODE_10W_DESKTOP
5

修改功耗模式。也可以通过 nvpmodel GUI （界面）设置

sudo /usr/sbin/nvpmodel -m <ModeID>

# 风扇配置

参考 Fan Profile Control

该框架实现了滞后，以防止风扇速度的频繁变化。例如，对于Jetson Xavier，当风扇配置文件设置为具有上述默认设置的“安静”时，框架会执行以下操作：

  • 温度达到 46 °C ，开风扇
  • 温度低于 38 °C ，关风扇 (46-8= Trip temperature - Hysteresis)

设置风扇模式有两种方式:

  1. 一种是如下的系统设置
  2. 还有一种是在 jtop 中手动设置（推荐）

查看当前风扇模式

sudo nvfancontrol -q

修改风扇配置流程如下：

  1. 停止风扇控制的系统服务
sudo systemctl stop nvfancontrol
  1. 设置默认风扇配置文件，将以下属性放入/etc/nvfancontrol.conf， <fan_profile> 修改为 quiet 或 cool.
FAN_DEFAULT_PROFILE <fan_profile>
  1. 移除状态文件
sudo rm /var/lib/nvfancontrol/status
  1. 启动风扇控制的系统服务
sudo systemctl start nvfancontrol

# jtop 监控软件

jtop 工具用于监控 Xavier

sudo apt install -y python3-pip
sudo python3 -m pip install jetson-stats  # 安装 jtop
sudo systemctl restart jtop.service       # 激活 jtop.service
sudo jtop                                 # 启动

修改设置，转到 6CTRL：

  • 按键 m/p 控制风扇转速
  • 按键 s 和 e 设置 jetson_clocks Running 和 boot Enable 进行风扇开机自启动
  • 按键 +/- 修改 NVP Model （功耗模式） 修改设置，转到 6CTRL：
  • 按键 m/p 控制风扇转速
  • 按键 s 和 e 设置 jetson_clocks Running 和 boot Enable 进行风扇开机自启动
  • 按键 +/- 修改 NVP Model （功耗模式）
