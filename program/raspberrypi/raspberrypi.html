# 树莓派

树莓派的官方文档: Raspberry Pi Documentation

# 安装系统

树莓派 4B 支持 64-bit

  • Ubuntu for Raspberry Pi: Ubuntu 官方支持的系统，对于硬件支持不一定非常完善，但是性能好（推荐，后续默认都是 ubuntu）
  • Raspberry Pi OS (64-bit): 官方原生支持的系统，对于硬件支持完善，但是性能拉胯，不然 32bit 的系统

烧写系统一律推荐使用 balenaEtcher

# 安装 Ubuntu for Raspberry Pi

在 Ubuntu 官方找到镜像文件:Ubuntu for Raspberry Pi，该网站是最新版本的 Ubuntu 24.04 ，如果希望选择旧版本，可以在 Ubuntu 22.04 LTS 找到

  • 如果需要桌面，并且可以通过显示器初始化，选择 ubuntu-22.04.4-preinstalled-desktop-arm64+raspi.img.xz，这种方式较为简单
  • 如果需要服务器版本，或者无显示器初始化，选择 ubuntu-22.04.4-preinstalled-server-arm64+raspi.img.xz，这种方式比较复杂，可以参考后续 “无显示器初始化” 教程

# 无显示器初始化

这种方式是有前提的，需要确保能通过路由器管理查看到树莓派的 IP 地址

# 配置无线网络

烧写完成后不要上电，否则配置无法生效，可能需要拔掉再插上主机

在主机上打开 system-boot 目录（可以 VSCode 打开），打开 network-config 文件，如果没有则创建并添加如下内容

# This file contains a netplan-compatible configuration which cloud-init
# will apply on first-boot. Please refer to the cloud-init documentation and
# the netplan reference for full details:
#
# https://cloudinit.readthedocs.io/
# https://netplan.io/reference
#
# Some additional examples are commented out below

version: 2
ethernets:
  eth0:
    dhcp4: true
    dhcp6: true
    optional: true
#wifis:
#  wlan0:
#    dhcp4: true
#    optional: true
#    access-points:
#      myhomewifi:
#        password: "S3kr1t"
#      myworkwifi:
#        password: "correct battery horse staple"
#      workssid:
#        auth:
#          key-management: eap
#          method: peap
#          identity: "me@example.com"
#          password: "passw0rd"
#          ca-certificate: /etc/my_ca.pem

在文件最后添加如下内容

wifis:
  wlan0:
    dhcp4: true
    dhcp6: true
    optional: true
    access-points:
      "xxx":
        password: "yyy"

其中, "xxx"是 WiFi 名称， "yyy"是密码，保留字符串引号。首次上电时会通过这些信息连接到网络，然后可以通过 SSH 连接到树莓派

如果有多个 WiFi 网络，可以添加多个 access-points，如下所示

wifis:
  wlan0:
    dhcp4: true
    dhcp6: true
    optional: true
    access-points:
      "xxx":
        password: "yyy"
      "aaa":
        password: "bbb"

# 启用 SSH

此外，还需要在 system-boot 目录下创建一个空文件 ssh，以启用 SSH

touch ssh

# 配置用户名

修改 user-data 文件，找到 chpasswd 部分，将 expire 改为 false，并添加用户名和密码

chpasswd:
  expire: true
  list:
  - ubuntu:ubuntu

ubuntu 留有一组默认的用户名，要求用户第一次登陆的时候改掉它

但是第一次接通电源的时候是无法连接到 WiFi 的，需要断电再上电。

此时通过 SSH 登陆会遇到问题 passwd: Authentication token manipulation error

因此, 我们需要使默认的用户名密码, 再坚持一会儿, 坚持到下一次登陆

# 设置用户名和密码

根据 Raspberry Pi OS Bullseye 的4月更新说明，经典的 pi 用户名和 raspberry 已经被取消，用户想要使用树莓派，需要连接显示器初始化树莓派，或者在 SD 卡中预先配置好用户信息。

这里使用第二种方法，在 SD 卡的启动分区中创建一个名为 userconf.txt 的文件，写入单行文本

<username>:<encrypted- password>

<username> 默认为 pi 即可，<encrypted- password> 为加密密码，可以使用 openssl 生成加密密码

echo 'mypassword' | openssl passwd -6 -stdin

提供一个成品方便没装 OpenSSL 的人，以下内容对应pi:raspberry

pi:$6$JMYKqZHSKkJ2eum7$UZ4oHrtIer2sGeZeiKI2gsgf.sI42LE.NNPN/FtYtQT5mIf0fZVKVNNkmt8dUBAIQFrKnSQZplTIcDYaWdTyM0

# 修改源

换源的目的是快速下载软件，如果网络顺畅能从原始源下载，则尽可能避免换源，以防一些软件无法下载

和 Linux 一样，树莓派的系统也是基于 Debian 的，所以可以使用 Debian 的源，所有的源可以在 Raspbian Mirrors 查到，清华源： Raspberrypi 软件仓库镜像使用帮助

Raspberry Pi OS (64-bit) 是 Debian 11 (Bullseye) 的一个分支，首先通过 uname -m 确定你使用的系统的架构，可以看到 aarch64，然后在 Raspbian Mirrors 找到对应的源，这里使用清华源，

编辑 /etc/apt/sources.list.d/raspi.list 文件

sudo cp /etc/apt/sources.list.d/raspi.list /etc/apt/sources.list.d/raspi.list.bak
sudo vi /etc/apt/sources.list.d/raspi.list

替换为以下内容

deb https://mirrors.tuna.tsinghua.edu.cn/raspbian/raspbian/ bullseye main non-free contrib rpi
# deb-src https://mirrors.tuna.tsinghua.edu.cn/raspbian/raspbian/ bullseye main non-free contrib rpi

deb [arch=arm64] https://mirrors.tuna.tsinghua.edu.cn/raspbian/multiarch/ bullseye main

更新源列表，并且顺便更新软件

sudo apt update

# 安装 ROS2

这里主要关注一个支持较长的 Humble Hawksbill (May 2022 - May 2027) 版本，

arm64 的树莓派安装不同系统支持不一样:

  • Raspberry Pi OS(Debian Bullseye (11)) 是 Tier 3 支持，可以通过 Docker 获得 Tier 1 支持
  • Ubuntu Jammy (22.04) 是 Tier 1 支持

Tier 1 支持的方法不再赘述，这里主要介绍 Tier 3 支持的方法

刷写操作系统后，请使用此链接 https://docs.docker.com/engine/install/debian/#install-using-the-convenience-script 安装Docker.

sudo apt  install -y curl
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

官方OSRF ROS 2 Docker容器定义可以在此处找到: https://github.com/osrf/docker_images/.

ROS

您可以选择使用ros-core、ros-base或ros-desktop。有关这些变体的更多信息，请参阅 https://github.com/osrf/docker_images/。

克隆 docker_images git仓库 到Raspberry Pi上，切换到上述链接的目录，然后进入您首选的变体所在的目录。
