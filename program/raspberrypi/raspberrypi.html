# 安装系统

树莓派 4B 支持 64-bit

  • Ubuntu for Raspberry Pi: Ubuntu 官方支持的系统，对于硬件支持不一定非常完善，但是性能好（推荐，后续默认都是 ubuntu）
  • Raspberry Pi OS (64-bit): 官方原生支持的系统，对于硬件支持完善，但是性能拉胯，不然 32bit 的系统

烧写系统一律推荐使用 balenaEtcher

# 设置用户名和密码

根据 Raspberry Pi OS Bullseye 的4月更新说明，经典的 pi 用户名和 raspberry 已经被取消，用户想要使用树莓派，需要连接显示器初始化树莓派，或者在 SD 卡中预先配置好用户信息。

这里使用第二种方法，在 SD 卡的启动分区中创建一个名为 userconf.txt 的文件，写入单行文本

<username>:<encrypted- password>

<username> 默认为 pi 即可，<encrypted- password> 为加密密码，可以使用 openssl 生成加密密码

echo 'mypassword' | openssl passwd -6 -stdin

提供一个成品方便没装 OpenSSL 的人，以下内容对应pi:raspberry

pi:$6$JMYKqZHSKkJ2eum7$UZ4oHrtIer2sGeZeiKI2gsgf.sI42LE.NNPN/FtYtQT5mIf0fZVKVNNkmt8dUBAIQFrKnSQZplTIcDYaWdTyM0

# 设置中文

sudo raspi-config

依次进入

  1. 5 Localisation Options Configure language and regional settings
  2. L1 Locale Configure language and regional settings
  3. 找到中文 zh_CN.UTF-8 UTF-8，按空格选中，然后按下空格打上 *，然后按下回车
sudo reboot

# 修改源

和 Linux 一样，树莓派的系统也是基于 Debian 的，所以可以使用 Debian 的源，所有的源可以在 Raspbian Mirrors 查到，清华源： Raspberrypi 软件仓库镜像使用帮助

Raspberry Pi OS (64-bit) 是 Debian 11 (Bullseye) 的一个分支，首先通过 uname -m 确定你使用的系统的架构，可以看到 aarch64，然后在 Raspbian Mirrors 找到对应的源，这里使用清华源，

编辑 /etc/apt/sources.list.d/raspi.list 文件

sudo cp /etc/apt/sources.list.d/raspi.list /etc/apt/sources.list.d/raspi.list.bak
sudo vi /etc/apt/sources.list.d/raspi.list

替换为以下内容

deb https://mirrors.tuna.tsinghua.edu.cn/raspbian/raspbian/ bullseye main non-free contrib rpi
# deb-src https://mirrors.tuna.tsinghua.edu.cn/raspbian/raspbian/ bullseye main non-free contrib rpi

deb [arch=arm64] https://mirrors.tuna.tsinghua.edu.cn/raspbian/multiarch/ bullseye main

更新源列表，并且顺便更新软件

sudo apt update

# 安装 ROS2

这里主要关注一个支持较长的 Humble Hawksbill (May 2022 - May 2027) 版本，

arm64 的树莓派安装不同系统支持不一样:

  • Raspberry Pi OS(Debian Bullseye (11)) 是 Tier 3 支持，可以通过 Docker 获得 Tier 1 支持
  • Ubuntu Jammy (22.04) 是 Tier 1 支持

Tier 1 支持的方法不再赘述，这里主要介绍 Tier 3 支持的方法

刷写操作系统后，请使用此链接 https://docs.docker.com/engine/install/debian/#install-using-the-convenience-script 安装Docker.

sudo apt  install -y curl
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

官方OSRF ROS 2 Docker容器定义可以在此处找到: https://github.com/osrf/docker_images/.

ROS

您可以选择使用ros-core、ros-base或ros-desktop。有关这些变体的更多信息，请参阅 https://github.com/osrf/docker_images/。

克隆 docker_images git仓库 到Raspberry Pi上，切换到上述链接的目录，然后进入您首选的变体所在的目录。
