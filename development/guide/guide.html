版本号 ​

版本号格式：主版本号.次版本号.修订号，版本号递增规则如下：

  1. 主版本号：当你做了不兼容的 API 修改时，递增。当功能模块有较大的变动，比如增加多个模块或者整体架构发生变化。一般适用重大变更的项目或发布新版本的时候。
  2. 次版本号：当你做了向下兼容的功能性新增时，递增。当功能模块有较大的变动，比如增加多个模块或者整体架构发生变化。一般适用重大变更的项目或发布新版本的时候。
  3. 修订号：当你做了向下兼容的问题修正时，递增。一般是 Bug 修复或是一些微小的变动，时间间隔不限，修复一个严重的bug即可发布一个修订版。
